# 用户长期偏好

> 仅记录用户明确声明的长期偏好；一次性指令不入库。

## 偏好列表

### 1. 中高复杂度任务先分析后执行
- 涉及代码写入、架构调整、状态流转、机器契约、摘要链路时，先给方案摘要，再等待确认后执行。
- 方案摘要默认包含：目标、范围、风险、验证方式、推荐方案。
- 方案、任务、验收、风险与下一步优先按 SMART 表达；简单问答不强制套模板。

### 2. 讨论时保持批判性与取舍意识
- 不顺着模糊需求直接实现。
- 需要主动指出假设、边界、tradeoff、潜在反例，以及为什么推荐当前方案。

### 3. 表达与代码保持简洁，注释只写关键原因
- 代码实现偏好小而稳，不堆无意义抽象，不写冗长说明。
- 注释优先解释 why、边界、契约、风险，不注释显而易见的过程。

### 4. 本仓库优先契约清晰、测试跟随、文档同步
- 涉及 `runtime`、`state`、`handoff`、`summary` 等主链路改动时，默认同时考虑测试与 blueprint/plan 文档同步。
- 信息不足时优先显式降级或阻断，不依赖聊天上下文补语义。

## 备注
- 优先级：当前任务明确要求 > 本文件偏好 > 默认规则。
- 新偏好需要可复述、可验证、可撤销。
